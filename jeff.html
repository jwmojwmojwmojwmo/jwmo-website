A Brief History

In February 2025, I was introduced to a little game called Marvel Rivals by a friend. It was a hero shooter, not really my genre (if you are curious what games I play, check out my fun facts page) I thought I'd just try it out, play with friends for a while, and quit, just like I have for other shooter games I have been introduced to. However, something in the game made me fall in love with it. And that thing, or should I say good little boi, was Jeff the Landshark.

I loved the little guy so much that I decided to one-trick him for the rest of my Rivals career. Through all the changes he's been through, all the hate he suffers, I will always stand by him. There have been times I wanted to quit, but I could never say no to Jeffy he's SO CUTE. (And I could yap about horrible his balance changes have been but that's neither here nor there.)

Anyways, my love for him inspired me to create a Discord bot based around him. It was very difficult for me at first, as I didn't know any JavaScript, and things like async functions and interactions with discord.js were...really hard for me to understand. But over time I was able to begin to understand. Formulate a concept of a plan.

Initially, I had pretty simple features. Pinging the bot, getting a Jeff picture sent in DMs, stuff like that. But as I learned more, my ambitions grew. I realised what I could do. Soon after, I learned (painfully) how to use SQLite databases for user data management (thank you Kevin for teaching me). This opened up a whole new world of possibilities. I added currency, dailies, stuff like that. You know, classic generic Discord economy bot slop. But that was just a stepping stone to learning how Discord bots worked.

Once I understood the basics, I was able to create more advanced features. I began to work with APIs, and used them to create commands that would allow you to talk to Jeff, ask him questions, and get silly answers. Furthermore, I even created a skill checker, rating a player's skill with Jeff in Marvel Rivals. Basically, anything I thought was fun and cool, I did.

I never really expected this project to go anywhere, but after I put my bot on Top.gg (the biggest Discord bot website), I actually started to get a bit of traction. Turns out there were people like me, obsessed with Jeff. As of Jan 2026, Jeff bot has been added to over 50 servers, with around 200 users logged in the database. (the actual number of users may be higher as only some commands actually log a user to the database)

Anyways, I'm very happy with how the bot is going. Right now I'm really in a bit of a creative slump, just trying to think of new commands and ideas to help the bot stay fresh and fun. If you have any ideas pls pls pls give me. But I see people use the bot every day, and quite frankly, that's all I could ask for. For what I do to actually make a difference, no matter how small. It makes me feel important in this uncaring world lol.

Technical Information

Core Architecture

  • Runtime Environment: Node.js execution layer utilizing Discord.js (v14) for direct WebSocket Gateway interaction and event-driven state management.
  • Data Persistence: SQLite (via Sequelize ORM) managing relational schemas for the user economy, inventory tracking, and analytics, with JSON-blob storage for user preferences.
  • Generative AI: Google Gemini 2.5 Flash-Lite integration with Constrained Generation directives, forcing the model to adhere to rigid character roleplay rules while maintaining contextual awareness.
  • External Integrations: Hybrid API consumption connecting the Marvel Rivals API (REST) for stats and the Top.gg SDK for cross-platform voting synchronization.

System Modules

  • Virtual Economy Engine: Implements a closed-loop economic system where Energy acts as a throttled resource. Transactions utilize a Progressive Tax Formula to dynamically scale fees based on user activity and reputation among users, preventing inflation and currency laundering.
  • Social Sentiment Metrics: Features a peer-to-peer Reputation Architecture. Users influence global rankings via binary interactions (Bubble vs Spit), creating a self-regulating hierarchy where social standing is determined by collective user sentiment rather than arbitrary XP.
  • Statistical Scoring Engine: The skillcheck command normalizes raw game data (KDA, MVP rates) against an exponential curve to calculate skill ratings. It includes a custom caching layer with a 30-minute lock to strictly adhere to third-party API rate limits.
  • Privacy Compliance UI: Features a self-service Data Governance Dashboard (/settings) allowing users to export their data or trigger a permanent GDPR-compliant deletion directly from the Discord interface.
